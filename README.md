# sloppatch 📜👉🏻🗑️

A patch tool designed with enhanced configuration options specifically to handle the quirks and potential inaccuracies of patches generated by Large Language Models (LLMs).

## Features

*   **Fuzzy Matching**: Allow searching for the correct hunk location within a specified number of lines above/below the target line number, accommodating potential line count errors by the LLM.
*   **Ignore context line case**: Optionally ignore case differences in context lines when searching for the patch location.
*   **Ignore whitespace differences**: Optionally ignore whitespace differences (tabs vs. spaces, leading/trailing spaces) when matching context lines.
*   **String trimming**: Optionally trim leading and trailing whitespace from lines before comparison.
*   **Flexible Empty Line Handling**: Configure how the parser treats completely empty lines within the patch.
*   **Skip malformed hunk lines**: Configure how to handle lines in the patch that don't conform to the expected format or appear out of sequence.

*Note: This patch utility can only be applied to a single file.*

## Motivation

This tool was created to address common issues when applying patches suggested by LLMs:

*   LLMs often incorrectly omit leading spaces (e.g., on empty lines), which causes standard patch tools to fail.
*   LLMs can make errors when counting line numbers, making precise offsets unreliable.
*   It's unnecessarily difficult to force an LLM to calculate the exact required hunk offset and length accurately every time.
*   LLMs sometimes inconsistently use spaces versus tabs or add/remove leading/trailing whitespace.

`sloppatch` provides configurable tolerance for these common LLM-generated patch variations.

## Format

1.  Each line in the patch begins with a special character:
    *   `#` -- Hunk header. Defines a "block" of changes. Must specify the starting line number in the target file.
    *   `=` -- Context line. Used to locate the correct position in the target file where the hunk should be applied.
    *   `-` -- Remove line. Indicates a line to be removed from the target file (context part).
    *   `+` -- Add line. Indicates a line to be added to the target file.
    *   `/` -- No newline marker. Indicates that the preceding `+` line should not have a newline character appended automatically.
2.  A patch must begin with a hunk header (`#`).
3.  A patch can contain multiple hunks.
4.  No newline marker (`/`) must be located after the `+` line. It is ignored if placed after other line types.

### Format Specification:

```
# <LINE START NUMBER> # <COMMENT (optional)>
=<CONTEXT (existing) LINE>
-<CONTEXT (existing) LINE TO DELETE>
+<NEW LINE TO ADD>
=<CONTEXT (existing) LINE>
```

### Example:

```patch
# 4 # First hunk
=line4
=line5
-line6
-line7
+line6 (new)
+line7 (new)
=line8
# 100 # Second hunk
=line100
=line101
+line102
```

# Installation

Install the latest version directly from GitHub:

```bash
pip install git+https://github.com/Toliak/sloppatch
```

To install a specific version (replace `v0.1.0` with the desired tag):

```bash
pip install git+https://github.com/Toliak/sloppatch@v0.1.0
```

## Development Setup

Clone the repository and set up a virtual environment:

```bash
git clone https://github.com/Toliak/sloppatch.git
cd sloppatch
python3 -m venv .venv
source ./.venv/bin/activate # On Windows: .\.venv\Scripts\activate
pip install -e '.[dev]'
```

### Using Dev Container (Optional)

If you have Docker and Make installed, you can use the provided dev container setup:

```bash
make docker-dev
```

## Running Linters and Tests

Ensure your development environment is activated.

Run Ruff linter:

```bash
make ruff
```

Run MyPy type checker:

```bash
make mypy
```

Run unit tests:

```bash
make test
```
